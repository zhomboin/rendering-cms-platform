# 后台管理系统设计规范（DESIGN.md）

## 1. 设计理念

* **简洁优先**：减少视觉干扰，突出核心数据
* **信息清晰**：优先保证数据可读性与结构层级
* **统一一致**：颜色、间距、组件保持一致性
* **现代优雅**：圆角 + 柔和阴影 + 低对比背景

---

## 2. 布局体系

### 整体结构

```
侧边栏（240px） + 主内容区
  ├── 顶部导航（80px）
  └── 内容区域（自适应）
```

### 栅格规则

* 基础间距单位：**8px**
* 卡片间距：**20px ~ 24px**
* 内容区域：自适应布局（响应式）

---

## 3. 颜色体系

### 主色

* Indigo：`#4F46E5`

### 语义色

* 成功：`#22C55E`
* 警告：`#F59E0B`
* 错误：`#EF4444`

### 中性色

* 页面背景：`#F8FAFC`
* 卡片背景：`#FFFFFF`
* 边框：`#E2E8F0`
* 主文本：`#0F172A`
* 次文本：`#64748B`

### 使用原则

* 主色仅用于关键交互与强调信息
* 避免大面积高饱和色块

---

## 4. 字体规范

| 层级   | 字号   | 字重      | 用途   |
| ---- | ---- | ------- | ---- |
| 一级标题 | 24px | Bold    | 页面标题 |
| 二级标题 | 18px | Bold    | 模块标题 |
| 正文   | 14px | Regular | 内容文本 |
| 辅助文字 | 12px | Regular | 说明信息 |

字体建议：系统无衬线字体（如 Inter / PingFang / 微软雅黑）

---

## 5. 组件设计规范

### 5.1 卡片（Card）

* 圆角：`24px`
* 内边距：`20px`
* 阴影：`shadow-sm`
* 边框：浅色边框

### 5.2 按钮（Button）

| 类型   | 样式          |
| ---- | ----------- |
| 主按钮  | 主色背景 + 白色文字 |
| 次按钮  | 边框 + 浅背景    |
| 幽灵按钮 | 无背景，hover高亮 |

### 5.3 侧边栏（Sidebar）

* 激活状态：主色背景
* hover：浅灰背景
* 图标与文字间距：12px

### 5.4 表格（Table）

* 行高：48px
* hover：浅背景高亮
* 分隔线：细边框

---

## 6. 数据可视化规范

### 折线图

* 平滑曲线
* 双数据线（主色 + 成功色）
* 带渐变填充

### 柱状图

* 顶部圆角
* 单色主色系

---

## 7. 交互设计

### 状态定义

| 状态     | 表现        |
| ------ | --------- |
| Hover  | 背景轻微变化    |
| Active | 提升层级 + 主色 |
| Focus  | 描边或高亮     |

### 动效规范

* 动画时长：150ms ~ 250ms
* 动画曲线：ease-out

---

## 8. 可访问性（Accessibility）

* 对比度 ≥ 4.5
* 点击区域 ≥ 40px
* 图标需添加 aria-label

---

## 9. 工程实现映射

### 技术栈

* React 组件化开发
* TailwindCSS 原子化样式

### 设计 Token 示例

```
primary = #4F46E5
radius-lg = 24px
spacing-base = 8px
```

---

## 10. 可扩展方向

* 深色模式（Dark Mode）
* 主题定制（Theme System）
* 权限驱动 UI（RBAC UI）
* 移动端适配（Responsive）

---

## 11. 总结

本设计体系核心目标：

* 提升数据展示效率
* 降低用户认知负担
* 支持组件化扩展
* 可直接用于生产环境落地
